I'll help you view the properties of a PEM certificate using OpenSSL commands. Here are the most common ways to inspect a certificate:

1. View Basic Certificate Information:

`openssl x509 -in certificate.pem -text -noout`

This command will display:

- Subject (who the certificate is issued to)
- Issuer (Certificate Authority)
- Validity dates
- Public key information
- Signature algorithm
- Extensions

2. Check Specific Details:

## View Subject

`openssl x509 -in 7e-39-24-5a-b3-b0-57-d2-0a-8e-aa-59-57-31-04-23-3b-95-ad-91.pem -subject -noout`

## View Issuer

`openssl x509 -in 7e-39-24-5a-b3-b0-57-d2-0a-8e-aa-59-57-31-04-23-3b-95-ad-91.pem -issuer -noout`

## View Dates

`openssl x509 -in 7e-39-24-5a-b3-b0-57-d2-0a-8e-aa-59-57-31-04-23-3b-95-ad-91.pem -dates -noout`

## Verify Certificate

`openssl x509 -in 7e-39-24-5a-b3-b0-57-d2-0a-8e-aa-59-57-31-04-23-3b-95-ad-91.pem -checkend 0`

3. Verify Certificate Expiration:

## Check if Certificate Will Expire in next 30 Days

`openssl x509 -in 7e-39-24-5a-b3-b0-57-d2-0a-8e-aa-59-57-31-04-23-3b-95-ad-91.pem -checkend 2592000`

4. Get Fingerprint:

## MD5 Fingerprint

`openssl x509 -in 7e-39-24-5a-b3-b0-57-d2-0a-8e-aa-59-57-31-04-23-3b-95-ad-91.pem -fingerprint -noout`

## SHA1 Fingerprint

`openssl x509 -in 7e-39-24-5a-b3-b0-57-d2-0a-8e-aa-59-57-31-04-23-3b-95-ad-91.pem -fingerprint -sha1 -noout`

Key Tips:

- Replace certificate.pem with your actual certificate filename
- The -noout flag prevents printing the entire certificate
- Use -text for comprehensive details
- Ensure you have OpenSSL installed on your system

To validate a public-private key pair using OpenSSL, you can follow these steps:

1. **Validate the Private Key:**

2 Ensure that the private key is valid and correctly formatted. Run the following command on your private key file:

`openssl rsa -check -in private_key.pem`

2. **Extract the Public Key from the Private Key:**

2 Use the following command to extract the public key from the private key file. This will help in comparing it with your existing public key:

`openssl rsa -in private_key.pem -pubout -out extracted_public_key.pem`

2. **Compare the Public Keys:**

2 Compare the existing public key and the extracted public key to ensure they match:

`diff original_public_key.pem extracted_public_key.pem`

1. If there is no output from the diff command, it means the keys match.
2. **Verify using OpenSSL:**

2 Another method to verify the pair is using encryption and decryption:

▪ **Encrypt a Message with the Public Key:**

`echo "test message" | openssl rsautl -encrypt -pubin -inkey original_public_key.pem -out encrypted_message.bin`

▪ **Decrypt the Message with the Private Key:**

`openssl rsautl -decrypt -inkey private_key.pem -in encrypted_message.bin`

2. If the decrypted message matches the original "test message", the key pair is valid.
